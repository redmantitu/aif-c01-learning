DOMAIN 1 Fundamentals of AI & ML (Cheat Sheet included)

Exam Trigger Cheat Sheet (Memorize!)

Predict Category? (Spam, Fraud) Classification (Supervised)

Learn by Trial/Error? (Robot) Reinforcement Learning
Minimize False Negatives? (Cancer) Maximize RECALL
Minimize False Positives? (Spam) Maximize PRECISION

AI vs ML vs Deep Learning vs GenAI

  • Artificial Intelligence (AI): Any technique that enables computers to mimic human intelligence.
  • Machine Learning (ML): Subset of AI. Machines "learn" from data without being explicitly programmed.
  • Deep Learning (DL): Subset of ML. Uses Neural Networks with many layers (like the human brain).
  • Generative AI (GenAI): Subset of DL. Creates new content (Text, Images, Code) rather than just classifying existing data.
Supervised Learning

Labeled Data. You know the answer.
Ex: Forecasting sales (Regression), Spam detection (Classification).

Unsupervised Learning

Unlabeled Data. Finding hidden patterns.
Ex: Customer Segmentation (Clustering), Anomaly Detection.

Reinforcement Learning

Rewards & Penalties. Trial and error.
Ex: Robotics, Gaming, Self-driving cars.

1. The ML Lifecycle (Key Process)

Exam Tip: Memorize the order of these steps.

  1. Business Goal: Define the problem.
  2. Data Preparation: Collection, Cleaning, Labeling, Feature Engineering.
  3. Model Training: Selecting algorithm, splitting data (Train/Test/Validate).
  4. Model Evaluation: Testing against unseen data (Metrics).
  5. Deployment: Endpoint creation (Real-time vs Batch).
  6. Monitoring: Drift detection (Data Drift, Model Drift).

2. Evaluation Metrics Deep Dive

Metric Use Case
Recall / Sensitivity Minimize False Negatives.
Ex: Cancer diagnosis, Fraud detection (Catch ALL cases).
Precision Minimize False Positives.
Ex: Spam filter (Don't delete real mail).
F1 Score Harmonic mean. Use when data is Imbalanced.
ROUGE / BLEU / BERTScore GenAI Specific: Evaluating text summarization & translation quality.

DOMAIN 2 Generative AI & Amazon Bedrock

The Foundation Model Lifecycle (Exam Critical)

1. Pre-training Massive unlabeled data. Expensive. Learn patterns.
2. Adaptation Fine-tuning / RAG to add domain knowledge.
3. Evaluation Human vs Automated (ROUGE/BLEU) assessment.
4. Deployment Optimization (Quantization) & Hosting.

Controlling the Output (Inference Parameters)

Temperature

0 = Deterministic (Math/Code).
1 = Creative (Poetry).

Top-P (Nucleus)

Limits choices to cumulative probability P (e.g., top 90%). Cuts long tail.

Top-K

Hard limit. Choose from top K tokens only (e.g., top 50).

Stop Sequences

String that tells the model to stop generating.

Bedrock Components (The Core of the Exam)

Knowledge Bases (RAG)

Connects FM to your data sources. Workflow:

  1. Data Source (S3)
  2. Chunking: Breaking text into small pieces.
  3. Embedding: Titan Embeddings model converts text to vectors.
  4. Vector Database: Stores vectors (OpenSearch Serverless, Aurora, Pinecone, Redis, Neptune).
  5. Retrieval: Finds relevant chunks for the query.
Agents

Executes multi-step tasks.

  • Orchestration: Breaks down tasks (Reasoning).
  • Action Groups: Defines APIs the agent can call (OpenAPI schema).
  • Lambda Functions: The actual code execution.
  • Trace: Allows you to see the agent's "Chain of Thought" reasoning.
Bedrock Pricing Models
  • On-Demand: Pay-per-token (Input/Output). Good for fluctuating traffic.
  • Provisioned Throughput: Purchase "Model Units" for a term (1-6 months). Guaranteed performance. Required for fine-tuned models.
  • Batch Inference: Process large datasets offline (S3 bucket input/output). Cheaper, strictly non-real-time.

RAG (Retrieval Augmented Generation)

  • Goal: Connect to internal/new data.
  • Pros: Cheaper, no training required, citations enabled, fewer hallucinations.
  • Use Case: "Answer questions about our internal PDF policy docs."

Fine-Tuning

  • Goal: Change model behavior/style.
  • Pros: Teaches "industry speak" (Medical/Legal), specific formatting.
  • Use Case: "Summarize medical reports in a very specific format."

Service Confusion Buster (Common Exam Traps)

Service A Service B Difference
Amazon Kendra Amazon OpenSearch Kendra is an out-of-the-box intelligent enterprise search service (connectors included). OpenSearch is a database/search engine you build upon (often used as Vector DB).
Textract Rekognition Textract is for Documents (OCR, Tables, Forms). Rekognition is for Photos/Videos (Faces, Objects, Scenes).
Q Business Q Developer Q Business connects to enterprise data (Wiki, Docs) for employees. Q Developer lives in the IDE for coding and AWS Console for troubleshooting.
SageMaker Canvas SageMaker Studio Canvas is No-Code (Business Analyst). Studio is an IDE for Data Scientists (Python/Jupyter).

SageMaker Toolset (Memorize These)

  • SageMaker Canvas: No-code interface for business analysts.
  • SageMaker Data Wrangler: Prepares/cleans data for ML (300+ transformations).
  • SageMaker Autopilot: AutoML. Automatically builds/trains/tunes best model.

DOMAIN 4 Responsible AI (Principles)

The Pillars of Responsible AI

1. Fairness (Bias)

System should treat all groups of people equally.
Tool: SageMaker Clarify

2. Explainability

Understanding HOW and WHY a model made a decision.
Tool: SageMaker Clarify (SHAP) / Model Cards

3. Privacy & Security

Data protection throughout the lifecycle.
Tool: PrivateLink, KMS, Bedrock Data Protocol

4. Robustness

System remains safe even under attack (Prompt injection) or erroneous input.

Deep Dive: PII Protection

Scenario: You have a dataset in S3 with customer emails (PII). You need to clean it before training.

1. Discovery Use Amazon Macie to scan S3 and find the PII.
2. Redaction Use Amazon Comprehend or Glue DataBrew to mask/redact it.

Bias Deep Dive

  • Pre-training Bias: Exists in the raw data (internet data).
  • Selection Bias: Training data doesn't represent the real world population.
  • Overfitting: Model memorizes noise instead of patterns.

DOMAIN 5 Security, Compliance & Governance

Security & Compliance Tools

  • Amazon Macie

    Security service that uses ML to discover and protect sensitive data (PII) in Amazon S3.

  • AWS Config

    Audits configuration history. "Did the setting change?" (Compliance)

  • AWS CloudTrail

    Audits API activity. "Who called the API?" (Auditing)

  • AWS KMS

    Encryption keys. Encrypts training data and model artifacts at rest.

  • VPC Endpoints (PrivateLink)

    Keeps traffic between your VPC and AWS services (Bedrock/SageMaker) on the AWS PRIVATE network (not public internet).

Governance & Shared Responsibility

AWS Responsibility (Security OF the Cloud)
  • Physical Data Centers
  • The Compute/Storage infrastructure
  • The "Base" Foundation Models (Bedrock)
Customer Responsibility (Security IN the Cloud)
  • Your Data (Encryption)
  • Your Application (Front-end)
  • Preventing Prompt Injection
  • Verifying Model Output (Accuracy/Bias)
